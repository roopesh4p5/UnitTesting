4. Setup Angular Test

4.1. Installing Tools-Utilities

To create an Angular application and begin testing, follow these steps to install the necessary tools:

  • Install Node (includes NPM)
  • Install Angular CLI globally:
    npm install -g @angular/cli or npm i -g @angular/cli
  • Create a new Angular project:
    ng new project/appName or ng new angular-unit-test-demo
  • Run the Angular application locally:
    ng serve
  • Execute test cases:
    ng test

ng test runs all test files ending with .spec.ts and displays results via Karma.

Note:

No additional setup is required for unit testing thanks to Angular CLI, which configures Jasmine and Karma automatically.

For detailed setup instructions, refer to:

  • Angular CLI Getting Started
  • Angular 7 Step-by-Step

4.2. Core Package-Dependencies for Testing

The following dependencies are essential for Angular testing:

Images should be displayed inline or linked directly to provide visual context.

  • jasmine-core: Framework for creating tests (.spec files)
  • karma: Task runner tool for tests
  • protractor: End-to-End testing framework

4.3. Core Test Setting-Configuration Files

Key configuration files for Angular tests:

  • karma.conf.js: Unit test runner configuration
  • protractor.conf.js: End-to-End test framework configuration
  • test.ts: Main entry point for unit tests

The karma.conf.js file specifies the setup for Karma, using test.ts to load test files recursively.
