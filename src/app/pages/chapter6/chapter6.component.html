6.6.1. SetUp and TearDown

  • beforeAll(): Called once before all specs
  • afterAll(): Called once after all specs
  • beforeEach(): Called before each spec
  • afterEach(): Called after each spec

6.6.2. Triple A - AAA - 3A - Arrange, Act and Assert pattern/structure

  • Arrange: Initialize system under test
  • Act: Call the method/function to perform some activity
  • Assert: Check the expected outcome

6.9. Unit Testing Limitations

  • Routers (need integration testing)
  • Template interpolation and binding
  • Event Handlers
  • Can't catch every error in a program
  • Can't evaluate all execution paths
  • Focuses on a unit of code, can't catch integration errors or broad system-level errors
