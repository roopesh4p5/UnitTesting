5. Advanced Angular Testing Techniques

5.2. Creating a Spec Test File

Create a new test file with a .spec.ts extension for each component:

  • Pair each class/component file (e.g., header.component.ts) with its test file (e.g., header.component.spec.ts)
  • Keep test files alongside the files they test for easier maintenance
  • Use ng test to run test scripts via Angular CLI

5.3.2. Unit Test Code Example

// Example: Testing header component
describe('HeaderComponent', () => {
    it('should have header title', () => {
        let title = 'Welcome to Angular Testing';
        expect(title).toEqual('Welcome to Angular Testing');
    });
});

5.3.3. Simple Function to Test with Jasmine

// Function to test
function sayHello() {
    return 'Hello World!!!';
}

// Jasmine test
describe('Say Hello', () => {
    it('should say Hello World', () => {
        expect(sayHello()).toEqual('Hello World!!!');
    });
});

5.4. Widely Used Angular Unit Testing Functions

5.4.1. describe()

The describe() function defines a test suite, grouping related tests:

describe('ComponentName', () => {
    // Tests go here
});

5.4.2. it()

The it() function defines an individual test spec:

it('should perform a specific task', () => {
    // Test logic/assertions go here
});

5.4.3. expect()

The expect() function is used to create expectations/assertions:

expect(value).toBe(expected);
expect(value).toEqual(expected);
// Other matchers...

5.4.4. Various Matcher Functions

Jasmine provides various built-in matchers for different types of assertions:

expect(array).toContain(member);
expect(fn).toThrow(error);
expect(value).toBeGreaterThan(number);
// and more...
